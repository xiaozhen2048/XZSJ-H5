> 日期：2026-06-03  
> 状态：待审核

---

## 一、项目背景

考公博主为学员提供江苏行测刷题课程，当前使用飞书云文档作为前端展示课程列表和腾讯会议回放链接。现有问题：

- 飞书文档视觉简陋，分类不够清晰
- PDF 讲义散落在百度网盘
- 无法防止学员分享课程链接给非付费用户
- 移动端体验差

目标：构建一个移动优先的 H5 页面，具有激活码访问控制、清晰的课程分类、后台管理能力。

---

## 二、用户与规模

- **学员规模**：约 300 人
- **主要设备**：手机（微信内打开），兼顾平板和电脑
- **用户画像**：备考公务员的女生为主

```
┌──────────────────────────────────┐
│   学员端 H5 (纯静态 HTML/CSS/JS)   │
│   部署: Cloudflare Pages            │
│                                   │
│   锁屏页 → 首页 → 课程列表 → 胶囊   │
└──────────────┬───────────────────┘
               │ fetch API
┌──────────────▼───────────────────┐
│   简易后端 (Node.js)              │
│   部署: Railway / 免费额度         │
│                                   │
│   POST /api/activate   激活验证    │
│   GET  /api/courses    课程数据    │
│   POST /api/admin/*    管理接口    │
│                                   │
│   数据: JSON 文件存储              │
└──────────────┬───────────────────┘
               │
┌──────────────▼───────────────────┐
│   云存储 (未来)                    │
│   阿里云 OSS / 腾讯云 COS          │
│   PDF 讲义直接下载                 │
└──────────────────────────────────┘
```

### 为什么选这个架构

- 纯前端 H5 加载最快，手机上体验接近原生
- 300 人规模，Railway + Cloudflare Pages 免费额度完全够用
- JSON 文件做数据库，零配置，备份方便（复制粘贴即可）
- 未来可随时迁移到云存储或正式数据库

## 四、页面结构

### 4.1 锁屏页

**路径**：首次访问（无有效激活记录时）

```
┌────────────────────────┐
│                        │
│     📚 行测刷题课堂     │  ← 品牌名
│  江苏真题 · 系统刷题    │  ← 一句话介绍
│                        │
│  ┌──────────────────┐  │
│  │  输入激活码        │  │  ← 6位或自定义格式
│  └──────────────────┘  │
│  [ 激活进入 ]          │
│                        │
│  ── 或 ──              │
│                        │
│  [ 👀 先看看 ]         │  ← 预览模式（无链接无密码）
│                        │
└────────────────────────┘
```

**交互逻辑**：

1. 输入激活码 → 后端验证 → 首次使用则绑定设备指纹 + 记录 IP
2. 「先看看」→ 进入首页但所有链接和密码隐藏，课程结构可见
3. 已激活用户在有效期内免登录（localStorage 存 token）

## 五、激活码系统

### 5.1 激活流程

```
学员打开H5 → 锁屏页
                │
    ┌───────────┴───────────┐
    │                       │
    ▼                       ▼
输入激活码              点击"先看看"
    │                       │
    ▼                       ▼
后端验证             进入预览模式
    │              (无链接无密码)
    ├─ 码不存在 → 提示错误
    ├─ 码已被激活 → 检查设备/IP
    │   ├─ 同一设备 → 放行（用户换手机重新登录）
    │   └─ 不同设备 → 提示"已被使用，联系博主"
    └─ 码未使用 → 激活成功
                    │
                    ▼
              记录设备指纹 + IP + 时间
              返回 token（存 localStorage）
              跳转首页
```

### 5.2 激活码格式

`XZST-XXXX-XXXX`（XZST = 小镇刷题，博主品牌名），共 300+ 个。

### 5.3 设备指纹

使用浏览器 fingerprint（UserAgent + Screen + Timezone + Canvas hash），无需第三方库。

### 5.4 异常检测

后台展示每个激活码的使用记录：
- 激活时间、设备指纹、IP 地址
- 同一激活码在不同设备 / IP 上使用时标记为异常
- 博主可手动封禁/解封

## 七、后台管理

### 7.1 入口

`/admin` — 只有博主知道的管理密码（不同于激活码）。

### 7.2 功能

- **课程管理**：添加/编辑/删除课程、视频链接和密码，按科目分类
- **激活码管理**：批量生成激活码、查看使用状态、封禁异常码
- **操作即时生效**，学员端刷新即见

### 7.3 数据安全

- JSON 文件定期备份（博主手动下载即可）
- 管理密码哈希存储，不存明文

## 九、部署方案

| 组件 | 平台 | 预估成本 | 国内速度 |
|------|------|----------|----------|
| 前端 H5 | Cloudflare Pages | 免费 | ✅ 快（国内有 CDN 节点） |
| 后端 API | Railway | 免费（512MB RAM） | ✅ 可接受（验证仅一次） |
| 激活码数据 | JSON 文件 | 无需额外服务 | - |
| PDF 云存储 | 阿里云 OSS（未来） | ~几元/月 | ✅ 快 |

**为什么不用 GitHub Pages**：`github.io` 域名在国内不稳定，部分地区打不开。Cloudflare Pages 有京东云合作的国内节点，不需备案，免费，体验好很多。
